## 1. 개요

### 1.1 한 줄 정의
바쁜 차주 대신 대리기사가 차량을 픽업해서 제휴 공업사에 입고시키고, 정비 완료 후 다시 가져다주는 **차량 정비 대행 플랫폼**.

### 1.2 해결하려는 문제
1. **차량 소모품 교환** (엔진오일·미션오일·브레이크오일/패드·타이어 등)
   - 직장인은 주로 주말에 가능 → 사람 많고 1-2시간+ 대기, 외진 공업사 위치로 이동·대기 불편
2. **자동차 정기검사**
   - 신차 4년 후 2년 주기 의무, 평일 위주 운영, 주말 풀부킹으로 3-4개월 후 예약

### 1.3 기본 작동 흐름
```
회원가입·차량등록 → 공업사 선택 → 정비항목·스펙 선택
  → 일시 예약 → 대리기사 자동 배정
  → 픽업 → 입고 → 정비 → 반납
  → 차계부 갱신 + 다음 점검 시점 알림
```
각 단계마다 고객에게 알림(배달의민족 패턴), 차량 위치는 GPS 실시간 추적.

### 1.4 대상 정비 범위
- 엔진오일, 엔진 플러싱, 미션오일, 오일팬
- 브레이크 패드/오일, 디스크 연마
- 타이어, 휠얼라인먼트
- 썬팅, 랩핑, PPF
- 손세차 등 **단순 소모품 교환·경정비 한정**
- 자동차 정기검사 대행

## 2. 페르소나

| ID | 이름 | 역할 | 한 줄 |
|---|---|---|---|
| P1 | 차주 (Customer) | 서비스 신청자 | 시간 없는 직장인. 차 맡기고 정비 결과만 받고 싶음 |
| P2 | 대리기사 (Driver) | 차량 픽업·반납 | 기존 대리운전 기사. 주간 부수입원 모색 |
| P3 | 공업사 직원 (Shop Staff) | 정비 수행 | 워킹 손님 의존이 심한 동네 공업사 |

## 3. 앱별 기능 요구사항 (유저 스토리)

### 3.1 고객용 앱 (Customer App)

#### M1. 회원·차량 등록
- **US-C-01**: 차주로서, **휴대폰 인증으로 회원가입**해서 빠르게 서비스에 진입하고 싶다.
  - *수용기준*: 본인인증 완료 후 1분 안에 메인 화면 진입.
- **US-C-02**: 차주로서, **차량번호만 입력하면 차종·연식·스펙이 자동 조회**되어 입력 수고를 줄이고 싶다 (헤이딜러/보험사 패턴).
  - *수용기준*: 차량번호 입력 시 차종·연식·트림이 자동 표기되며, 사용자는 confirm만 누름. 자동 조회 실패 시 수동 선택 fallback 제공.

#### M2. 공업사 탐색·선택
- **US-C-03**: 차주로서, **지도에서 가까운 제휴 공업사 리스트**를 확인하고, **공업사까지 거리에 따른 대리비**를 미리 보고 싶다.
  - *수용기준*: 지도 + 리스트 토글 가능. 각 공업사 카드에 거리/예상 대리비/평점/제공 서비스가 표기됨.
- **US-C-04**: 차주로서, **고객이 지정한 미제휴 공업사**도 선택하고 싶다 (단, 이 경우 공업사 예약은 차주가 별도 진행).
  - *수용기준*: 지정 공업사 선택 시 "예약 별도 진행 필요" 안내가 노출되고, 본인이 확인 후 진행.

#### M3. 정비 항목 선택
- **US-C-05**: 차주로서, **필요한 정비 서비스를 선택하면 차량에 맞는 추천 스펙**(예: 엔진오일 0W30 등)이 자동으로 안내되었으면 좋겠다.
  - *수용기준*: 정비 항목 선택 시 차량 정보 기반 추천 1순위 제품 + 호환 가능 제품 리스트 노출.
- **US-C-06**: 차주로서, **리스트에 없는 제품을 사용**하고 싶을 때 요청사항을 직접 적을 수 있어야 한다.
  - *수용기준*: 자유입력 필드 제공. 단, 공업사 재고 부족 시 **품절 안내 + 대체품 동의 여부**를 확인하고, 동의하면 그대로 진행, 미동의 시 예약 취소.

#### M4. 예약·결제
- **US-C-07**: 차주로서, **원하는 날짜·시간을 캘린더에서 선택**해 예약하고 싶다.
  - *수용기준*: 공업사가 가용한 시간대만 활성화되어 표기됨. 예약 확정 시 공업사로부터 응답 수취.
- **US-C-08**: 차주로서, 예약 시 **결제 금액의 구성**(정비비 + 대리비)이 명확히 보였으면 좋겠다.
  - *수용기준*: 결제 화면에 정비비·대리비·총액 분리 표기.

#### M5. 실시간 추적
- **US-C-09**: 차주로서, **배정된 대리기사 정보**(이름·평점·차량)를 미리 확인하고 싶다 (카카오T 유사).
  - *수용기준*: 배정 즉시 푸시 알림 + 기사 카드(이름·평점·연락 가능 버튼) 노출.
- **US-C-10**: 차주로서, **서비스 당일 차량의 현재 위치를 GPS로 실시간 추적**하고 싶다.
  - *수용기준*: 픽업 시점부터 반납 시점까지 지도 위에 차량 마커가 5초 간격으로 갱신됨.
- **US-C-11**: 차주로서, **차량 인수 시·출발 시·공업사 도착 시·정비 완료 시·반납 출발 시** 단계별 알림을 받고 싶다 (배달의민족 패턴).
  - *수용기준*: 5단계 모두 푸시 알림 또는 알림톡으로 자동 발송.

#### M6. 차계부·다음 점검 알림
- **US-C-12**: 차주로서, **정비 완료 후 정비 내역이 차계부 형식으로 자동 누적**되었으면 한다.
  - *수용기준*: 정비 완료 시점에 "차량 누적 주행거리 + 정비 항목"이 차계부에 한 줄 추가됨.
- **US-C-13**: 차주로서, **다음 점검 시점이 도래하면 알림**을 받고 싶다 (주행거리 기준 또는 날짜 기준 중 선택).
  - *수용기준*: 회원 설정에서 알림 기준(km vs 개월)을 선택. 도래 시점에 푸시 + 알림톡.

> ℹ️ 차계부 데이터의 활용(헤이딜러 등 중고차 업체 판매)은 별도 동의 항목으로 회원가입·마이페이지에서 관리. (자세한 내용은 §5 참조)

### 3.2 기사용 앱 (Driver App)

- **US-D-01**: 대리기사로서, **콜이 떴을 때 픽업 위치·공업사 위치·예상 수익**을 확인하고 수락 여부를 결정하고 싶다.
  - *수용기준*: 일반 대리운전 앱과 동일한 콜 수신 메커니즘. 30초 내 미응답 시 다음 기사에게 패스.
- **US-D-02**: 대리기사로서, **차량 인수 시 외관 사진을 표준 포맷에 따라 촬영·업로드**해야 한다 (차주 미동승 분쟁 대비).
  - *수용기준*: 앱이 부위별 가이드(전·후·좌·우·휠·실내·계기판)를 오버레이로 표시. 모든 부위 촬영 완료 전에는 출발 버튼 비활성화.
  - *⚠️ 미해결*: **촬영 부위·장수 표준은 별도 합의 필요** (§7 미해결 이슈 참조).
- **US-D-03**: 대리기사로서, **반납 시점에도 동일한 외관 사진을 촬영**해 사고·파손 여부를 기록해야 한다.
  - *수용기준*: 반납 사진 업로드 완료 후에만 운행 종료 처리됨.
- **US-D-04**: 대리기사로서, **운행 중 GPS 위치가 자동으로 송신**되어야 한다 (고객 화면에 표시됨).
  - *수용기준*: 백그라운드 위치 권한 동의 후 5초 간격 송신. 권한 미동의 시 운행 시작 불가.

### 3.3 공업사용 앱 (Shop App)

- **US-S-01**: 공업사로서, **들어온 예약 현황을 한눈에 확인**하고 고객에게도 노출되도록 관리하고 싶다.
  - *수용기준*: 일/주 단위 예약 캘린더 뷰. 신규 예약 도착 시 알림. 수락/거절 처리 가능.
- **US-S-02**: 공업사로서, **작업 내용을 단계별로 업데이트**(접수 → 작업 중 → 완료)할 수 있어야 한다.
  - *수용기준*: 각 단계 변경이 고객 알림으로 즉시 반영됨.
- **US-S-03**: 공업사로서, **특이사항(예: 추가 정비 발견)을 직접 수기로 입력**할 수 있는 탭이 필요하다.
  - *수용기준*: 자유입력 텍스트 필드. 입력 시 고객에게 알림과 함께 동의 요청.
- **US-S-04**: 공업사로서, **선택된 작업별 예상 소요시간**을 지정해 출고 시점을 예측 가능해야 한다.
  - *수용기준*: 작업 항목별 소요시간 입력. 합산 결과가 예상 출고 시각으로 노출.
- **US-S-05**: 공업사로서, **작업 완료 시 출고기사를 호출**할 수 있어야 한다 (입고기사와 출고기사가 다를 수 있음).
  - *수용기준*: "출고기사 콜" 버튼 누르면 운영자 또는 가용 기사에게 자동 배차 요청 발송.

### 3.4 차량용 앱 (Vehicle App) — ⚠️ 실현성 별도 검증 필요

- **US-V-01**: 차량 시스템에서, **누적 주행거리를 자동으로 송신**해 차계부가 별도 입력 없이 갱신되도록 한다.
  - *수용기준*: 차량 인포테인먼트 또는 OBD-II 동글에서 일 1회 이상 주행거리 송신.
- **US-V-02**: 차량 시스템에서, **수신된 주행거리를 기반으로 다음 정비 시점을 차주에게 알림**한다.
  - *수용기준*: 누적 주행거리가 정비 임계치(예: 엔진오일 7,000km) 도달 시 차주 푸시 발송.

> ⚠️ **미해결**: 차량 데이터 수집 방법(OBD-II 동글 vs 제조사 커넥티드 API vs CarPlay/Android Auto)은 별도 검증 필요. 제조사 API(현대 블루링크·기아 UVO 등)는 외부 개발자 공개가 제한적이어서, 초기에는 **공업사 입고 시 주행거리 수기 입력**으로 차계부를 채우는 것이 현실적 대안. (§5 외부 연동 참조)

### 3.5 관리자 앱 (Admin Web)

- **US-A-01**: 운영자로서, **1일 누적 결제 건수와 매출 현황**을 한눈에 확인하고 싶다.
  - *수용기준*: 일·주·월 단위 매출 대시보드. 정비비/대리비 분리 표기.
- **US-A-02**: 운영자로서, **고객 컴플레인과 Q&A 게시판**에 접근하고 답변할 수 있어야 한다.
  - *수용기준*: 컴플레인 티켓 리스트 + 상태 관리(접수/처리중/완료) + 답변 작성.
- **US-A-03**: 운영자로서, **공지사항을 등록·노출**할 수 있어야 한다.
  - *수용기준*: 공지 작성 후 고객 앱 메인에 노출 + 푸시 발송 옵션.
- **US-A-04**: 운영자로서, **신규 제휴 공업사를 등록**하고 계정을 발급할 수 있어야 한다.
  - *수용기준*: 공업사 정보(상호·사업자등록번호·주소·제공 서비스·평점 시작값) 입력 → 공업사 계정 발급.

## 4. 사용자 여정 (요약 3단계)

### J1. 첫 예약
회원가입 → 차량번호 등록·자동조회 → 공업사 선택 → 정비 항목·스펙 선택 → 캘린더에서 일시 선택 → 결제 → 예약 확정

### J2. 서비스 당일
배차 알림 → 기사 GPS 추적 시작 → 픽업 사진 업로드 → 출발 → 공업사 입고 → 정비 진행(공업사 단계 업데이트) → 정비 완료 → 출고기사 콜 → 반납 사진 업로드 → 반납 → 평점

### J3. 차계부 → 다음 점검 → 재예약
정비 내역 자동 누적 → 다음 점검 시점 도래 → 알림 → 재예약 (J1으로 회귀)

## 5. 외부 시스템 연동 (요약)

> 사전 리서치 결과 1차 후보. 본 SRS는 **연동 인터페이스 존재**만 명시하고, 구체 벤더 선정은 별도 의사결정.

| 영역 | 1차 후보 | 비고 |
|---|---|---|
| 차량번호 → 차량정보 | 자동차365 공공데이터 + 보강 벤더(예: 카비) / 옵션: 보험개발원 카히스토리 | 무료 + B2B 유료 옵션 |
| 지도·POI | 카카오맵(B2C 화면) / TMap(기사 ETA) | 카카오T대리·티맵대리 표준 패턴 |
| 결제 PG | 포트원(PortOne) + 토스페이먼츠 | 멀티 PG 추상화 |
| 푸시 알림 | FCM(앱 푸시) + 솔라피(알림톡·SMS) | 알림톡 도달율 88-90% |
| 사진 저장 | Cloudflare R2 + Cloudflare Images | egress 무료 |
| 본인인증 | NICE 본인확인 | 한국 표준 |
| 번호판/계기판 OCR | 네이버 CLOVA OCR | 입력 보조용 |
| 차량 데이터 (Phase 3) | OBD-II 동글 또는 제조사 커넥티드 API | 실현성 별도 검증 |

## 6. 제약·리스크 (요약)

> 제안자가 docx에 이미 제기한 리스크 + 사전 법무 리서치 결과를 1줄씩 통합.

| 영역 | 핵심 제약·리스크 | 권장 대응 |
|---|---|---|
| **차주 미동승 차량 운송** | 여객자동차법상 "대리운전"보다 "**탁송**"으로 분류하는 것이 안전 | 약관·서비스명에서 "탁송" 명문화 + 국토부 사전 유권해석 권장 |
| **사고 시 책임** | 표준 대리운전 LP보험은 차주 동승 전제 → 미동승 시 보장 공백 가능 | **탁송업자 종합보험**(현대해상·KB손해보험) 가입 + 고가차 단기 특약 자동 인수 |
| **차량 인수인계** | 분실·파손 시 사진·메타데이터 미보유 시 사업자 입증책임 | 표준 촬영 부위·장수 + EXIF·해시 보존 + 디지털 서명 |
| **차량 내 도난** | 약관규제법상 "고의·중과실 면책" 조항은 무효 | "귀중품 사전 신고제" 또는 "경과실 면책" 약관 한정 |
| **개인정보 (차량번호·GPS)** | 차량번호+소유자 결합 시 개인정보 / GPS는 위치정보법 적용 | **위치정보사업자 신고**(방통위) + 계층화 동의(필수/마케팅/제3자 제공) |
| **차계부 데이터 판매** | 차량 단위 식별 판매는 **별도 동의 필수** (무동의 시 형사처벌) | 가명정보 처리체계(개보법 §28-2) 구축 + 동의 UX 마련 |
| **정기검사 대행** | 자동차관리법상 위임 가능 | 표준 위임장 + 검사 결과 자동 통지 |
| **전자상거래 책임 분리** | 정비=중개 / 픽업·반납=직접제공 → 책임 이원화 필요 | 약관 분리 + "통신판매중개자 고지" 명문화 |

## 7. 부록

### 7.1 용어집

| 용어 | 설명 |
|---|---|
| **탁송** | 차주가 동승하지 않고 차량만 목적지로 이동시키는 행위. 대리운전과 법적 구분이 다름 |
| **대리운전** | 차주의 요청으로 차주의 차량을 운전해주는 용역 (보통 차주 동승 전제) |
| **LP보험** | 대리운전자보험. 대리운전 중 발생한 사고를 보장하는 의무·권장 보험 |
| **자동차 정기검사** | 신차 4년 후 2년 주기로 한국교통안전공단(TS)에서 받아야 하는 의무검사 |
| **PPF** | Paint Protection Film. 차량 외장 보호 필름 |
| **휠얼라인먼트** | 바퀴 정렬 정비. 핸들 쏠림·타이어 편마모 보정 |
| **GPS 추적** | 차량의 실시간 위치를 위성 신호로 받아 지도에 표시하는 기능 |
| **OBD-II** | 차량 내부 진단 포트. 동글을 꽂아 주행거리·연료 등 데이터 수집 가능 |
| **FCM / APNs** | 앱 푸시 알림 표준 (각각 안드로이드·iOS) |
| **알림톡** | 카카오톡으로 발송되는 비즈니스 메시지. 앱 미설치 사용자에게도 도달 |

### 7.2 미해결 오픈 이슈

> 미팅에서 제안자와 합의하거나, 별도 검증이 필요한 항목.

**docx에서 제안자가 이미 던진 질문**
- 공업사까지 거리에 따른 **대리비 요금표** — 시장 자료 확인 필요
- 차량 외관 사진 **촬영 부위·장수 표준** 합의 (탁송 업계 통상 8-17장)
- 주행거리 데이터 수집 방법 — **차량용 앱 실현성** 검증 필요

**사전 리서치에서 도출된 추가 결정 필요 사항**
- 서비스 분류를 **"탁송"** vs "대리운전" 중 어떻게 약관에 명문화할 것인가 (법적 리스크 가장 큰 의사결정)
- **데이터 판매(헤이딜러·엔카) 사업화 시점·범위** — 차량 단위 식별 vs 통계 단위 가명정보 (둘 중 어느 모델로 갈지)
- 초기 권역(예: 서울 강남구 vs 수도권 전체)과 **제휴 공업사 수** — 운영 인력·마케팅 비용에 직결
- 차량용 앱·OBD 동글 통합 여부 — Phase 1 포함 vs Phase 3 이연 (수억 단위 비용 차이)
